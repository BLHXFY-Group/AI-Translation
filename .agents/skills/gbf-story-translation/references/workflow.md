# 工作流

## 连续供稿

* 每个新会话任务按当前本地时间生成一次 `YYYYMMDD-HHMMSS` 格式的 `task_id`。依次提供的文件视为同一连续任务，始终沿用该值；不维护 current-task，不复用已结束任务的 ID。
* 当前任务目录固定为 `.codex-tmp/gbf-translation/<task_id>/`，其中保留术语、backlog、state、报告和原始副本。当前文件完成后等待下一文件；只有用户明确表示全部提供完毕、这是最后一个文件或要求最终检查时，才进入最终复查。
* `state.json`、原始副本和报告由脚本维护，不要手写或修改。`state.json` 保存同一个 `task_id`，脚本必须校验它与 `--task-id` 一致。
* 连续任务的第一个文件开始前完整读取一次“翻译与审校规则”；上下文经过压缩、摘要化或重建后，在继续任务前重新完整读取一次。除此之外不自动重读。
* `<任务目录>/terms.md` 顶部固定使用以下提醒，不得删除或改写：

```markdown
> 每文件校准：以当前日文和必要上下文为准，不增补原文没有的信息；保持当前术语、人物关系与语气，按自然中文完成审校。
```

其余内容作为最小前向约束，只保存后续文件仍可能使用且容易漂移的信息：用户指定译法、任务特有或存在歧义的专名、非显然的称呼与角色口吻，以及直接影响下一文件理解的未结束状态。

不记录译法无歧义且可从当前正文直接判断的人名和基础世界观词、普通词典词、已结束剧情摘要、处理历史或文件顺序。不得为判断是否需要记录而扩大搜索范围。每个文件完成后删除已经失去前向作用的内容，合并重复条目，不累积逐章日志。

## 正文读取

```text
python -B <Skill目录>/scripts/read_translation_csv.py <CSV>
```

脚本按 CSV 原顺序输出格式与记录统计、全部有效正文和 `译者` 记录；纯控制记录不输出。`译者` 不计入有效正文和空译统计。

## 更新 JSON 与写回

只生成 `<任务目录>/updates.json`：

```json
{
  "version": 1,
  "files": [
    {
      "path": "story/example/scene_evt000000_cp1_q1_s10.csv",
      "updates": {
        "1": "译文",
        "2": "第一行\\n第二行"
      }
    }
  ]
}
```

每个文件只提供仓库相对 `path` 和字符串 `ID → trans` 的 `updates`；没有改动时使用 `{}`。显示换行在 JSON 中写成 `\\n`。然后运行：

```text
python -B <Skill目录>/scripts/apply_translation_updates.py --task-id <task_id>
```

脚本完成备份、写回、确定性验证和 state 更新。失败时按错误修正 JSON 或译文后重试，不得绕过脚本。

## 待回溯记录

后续文件为前文提供新依据时，使用 `apply_patch` 更新 `<任务目录>/backlog.md`：

```text
原文件 | ID | 问题 | 新依据 | 建议修正 | 状态
```

状态只用 `待处理`、`已修正`、`仅报告`。最终复查时统一处理；无法确认或不应修改的项目标为 `仅报告` 并说明原因，不得静默删除。缺失文件、顺序异常和等待后文提供依据的事项也记录在 backlog，不写入 `terms.md`。

## 验证

日常写回已经包含确定性验证。只需独立比较时使用：

```text
python -B <Skill目录>/scripts/validate_translation_csv.py --task-id <task_id> --before <原文件或目录> --after <编辑后文件或目录>
```

连续任务最终复查使用：

```text
python -B <Skill目录>/scripts/validate_translation_csv.py --task-id <task_id>
```

脚本检查 CSV 结构、非 `trans` 字段、格式、真实换行、显示行数、引号和标签。

* `ERROR`：修正更新 JSON 或译文并重新运行，不得修改原始副本或绕过验证。
* `WARN`：人工判断空译、日文残留、长文本、标点、既有引号类型和既有标签，不得自动批量修复。
* 退出码 `0` 可带警告；退出码 `1` 表示未通过。只有需要定位省略详情时才读取完整报告。

## 完成条件

当前文件完成必须满足：所有目标正文已逐条处理并通过写回脚本登记；脚本无错误，警告已人工判断，术语和 backlog 已更新。没有修改时使用空 `updates`。

连续任务完成必须满足：

* 用户明确表示文件已全部提供，所有文件均已完成。
* backlog 均为 `已修正` 或 `仅报告`，受影响文件已重新写回。
* state 最终验证通过，所有警告已人工判断。
* 报告处理文件、修改数量、警告和未修改问题，然后只删除 `.codex-tmp/gbf-translation/<task_id>/`。
